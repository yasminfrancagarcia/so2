Neste trabalhp, desenvolvi o que foi pedido em cada uma das partes, separadamente 
em cada pasta contida em t2. 
Na parte 2, está implementado o escalonador mais simples, para testar ele basta compilar a pasta.
Para a parte 3, os 2 tipos de escalonadores estão separados na pasta "prio" para o de prioridade e  "roundRobin" para
o round robin. Para testar, apenas compilar as pastas também. 
A pasta "prio" seria a versão final do trabalho, mas apenas com o escalonador de prioridade. 
 

Para compilar, use o comando make. Para rodar, digite ./main.

Os arquivos so.c, so.h, processo.c, processo.h, lista.c, lista.h, metricas.c e metricas.h contêm a maior parte das modificações realizadas.
